## Detailed Setup

### 1. Client Inbox (Gmail)
- **Already connected**: kathabooth@gmail.com
- **Add**: Canned responses (Settings → Advanced → Enable templates)
- **Add**: Labels per client/event + filters for auto-sorting
- **Risk**: Personal domain (`@gmail.com`) affects perceived professionalism, but acceptable for short transition. Deliverability is fine for transactional email.

### 2. Invoices + Payments — Wave
- **Why**: True free accounting + invoicing, no monthly fee, unlimited invoices
- **Setup**: Create Wave account → connect Stripe for payments (Wave handles the integration)
- **Flow**: Send invoice via Wave → client pays online → auto-recorded in Wave ledger
- **Risk**: Transaction fees (2.9% + $0.60) unavoidable on any free tier. No contracts module.

### 3. Contracts + Signatures — SignRequest
- **Why**: 10 docs/month free, legally binding e-signatures, audit trail
- **Setup**: Create SignRequest account → upload PDF contract → add signature fields → send
- **Flow**: Client signs → both parties receive signed PDF → store in Drive
- **Alternative**: **Jotform Sign** (10 docs/month, easier UI)

### 4. Contracts — Document Drafting
- Use Google Docs for contract templates → export to PDF → send via SignRequest

## Concrete Workflow

### New Client Onboarding (per client):

```
1. EMAIL: Client inquiry → kathabooth@gmail.com
   → Label: "Client/[Name]" + canned response template for quote

2. INVOICE: Create in Wave
   → Email invoice via Wave (links to Stripe payment page)
   → Payment auto-recorded

3. CONTRACT: Draft in Google Docs (from template)
   → Export to PDF
   → Upload to SignRequest → add signature fields
   → Send to client
   → Download signed copy → save to Drive: /Clients/[Name]/Contract_Signed.pdf

4. TRACKING: Update Google Sheets tracker
   → Row: Client | Invoice Status | Contract Status | Payment Status | Event Date
```

## Legal Notes

1. **E-signature legality**: SignRequest complies with ESIGN Act and eIDAS — signatures are legally binding for business contracts in US/EU.

2. **Contract storage**: Keep signed PDFs in Drive with clear naming (`Contract_Signed_2026-07-10_ClientName.pdf`). This satisfies most record-keeping requirements.

3. **Payment records**: Wave maintains transaction history. Export monthly for backup.

4. **Terms of Service**: Ensure contract includes: "Electronic signature constitutes binding agreement per ESIGN Act."

## Migration Path to pb-v3

When pb-v3 ships:
- **Invoices**: Export Wave CSV → import to pb-v3 Supabase
- **Contracts**: Move PDFs from Drive → pb-v3 storage (Supabase Storage)
- **Client data**: Export Google Sheets CSV → import to pb-v3 clients table
- **Email**: Continue using Gmail until pb-v3 email (Resend) is ready, then migrate
